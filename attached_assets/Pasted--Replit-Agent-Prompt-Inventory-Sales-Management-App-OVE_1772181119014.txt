# Replit Agent Prompt — Inventory & Sales Management App

## OVERVIEW

Build a cross-platform mobile inventory and sales management app for a small retail business. The app runs on Android and iOS. It tracks stock, daily sales, orders, and generates reports. The primary input method is camera (scan labels, barcodes, shelves) powered by Claude AI vision API. Secondary input is voice and manual entry.

## TECH STACK (use exactly these, no substitutions)

- **Framework:** React Native with Expo (managed workflow, SDK 51+)
- **Navigation:** expo-router (file-based routing with bottom tabs)
- **Database:** Supabase (PostgreSQL + Auth + Storage for photos)
- **Offline cache:** expo-sqlite for offline mode, sync to Supabase when online
- **Camera:** expo-camera (photos + barcode scanning in ONE unified scanner screen)
- **OCR/AI:** Claude API (claude-sonnet-4-5-20250929 with vision) — called from backend only, never from client
- **Voice:** expo-speech-recognition for voice-to-text on Orders screen
- **Excel export:** xlsx library (SheetJS)
- **PDF export:** expo-print + expo-sharing
- **Charts:** victory-native for sales analytics
- **State management:** zustand
- **Styling:** nativewind (Tailwind for React Native)

## DATABASE SCHEMA (Supabase PostgreSQL)

```sql
-- Product categories
CREATE TABLE categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Main products table (stock)
CREATE TABLE products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  name TEXT NOT NULL,
  brand TEXT,
  serial_number TEXT,
  barcode TEXT,
  category_id UUID REFERENCES categories(id),
  quantity INTEGER DEFAULT 0,
  min_quantity INTEGER DEFAULT 2,
  netto_price DECIMAL(10,2),
  brutto_price DECIMAL(10,2),
  photo_url TEXT,
  notes TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);

-- Price history for tracking changes
CREATE TABLE price_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id UUID REFERENCES products(id) ON DELETE CASCADE,
  netto_price DECIMAL(10,2),
  brutto_price DECIMAL(10,2),
  changed_at TIMESTAMPTZ DEFAULT now()
);

-- Daily work sessions
CREATE TABLE work_days (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  date DATE NOT NULL,
  started_at TIMESTAMPTZ DEFAULT now(),
  ended_at TIMESTAMPTZ,
  is_closed BOOLEAN DEFAULT false,
  total_sold_count INTEGER DEFAULT 0,
  total_netto DECIMAL(10,2) DEFAULT 0,
  total_brutto DECIMAL(10,2) DEFAULT 0,
  total_profit DECIMAL(10,2) DEFAULT 0,
  notes TEXT
);

-- Sales records
CREATE TABLE sales (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  work_day_id UUID REFERENCES work_days(id),
  product_id UUID REFERENCES products(id),
  product_name TEXT NOT NULL,
  product_brand TEXT,
  quantity INTEGER DEFAULT 1,
  purchase_netto DECIMAL(10,2),
  purchase_brutto DECIMAL(10,2),
  sale_netto DECIMAL(10,2),
  sale_brutto DECIMAL(10,2),
  profit DECIMAL(10,2),
  sold_at TIMESTAMPTZ DEFAULT now()
);

-- Orders for next day
CREATE TABLE orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  work_day_id UUID REFERENCES work_days(id),
  product_name TEXT NOT NULL,
  product_brand TEXT,
  quantity INTEGER DEFAULT 1,
  expected_netto DECIMAL(10,2),
  expected_brutto DECIMAL(10,2),
  supplier TEXT,
  status TEXT DEFAULT 'ordered' CHECK (status IN ('ordered', 'received', 'partial', 'cancelled')),
  product_id UUID REFERENCES products(id),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## SCREEN-BY-SCREEN SPECIFICATIONS

### 1. UNIFIED SCANNER SCREEN (`scan.tsx` and `scanner/index.tsx`)

THIS IS THE MOST IMPORTANT SCREEN. It must handle ALL scanning scenarios through ONE interface.

**How it works:**

- Full screen camera view with a semi-transparent overlay
- At the bottom: 3 mode toggle buttons: “Barcode” | “Label” | “Shelf”
- Above the buttons: context selector: “Add to Stock” | “Sell” (two big toggle buttons)
- The scanner is opened from multiple places but always looks the same

**Barcode mode:**

- Uses expo-camera barcode scanner (onBarCodeScanned)
- Scans EAN-13, EAN-8, UPC-A, UPC-E, Code128, QR
- On scan: vibrate phone, search database for matching barcode
- If found: show product card overlay with name, brand, photo, current quantity, price
  - If context is “Add to Stock”: show quantity input (+/- buttons, default 1), confirm button → adds to stock quantity
  - If context is “Sell”: show quantity input (default 1), sale price fields (netto/brutto, pre-filled from product data), confirm → creates sale record, decrements stock
- If NOT found: prompt “Product not in database. Take label photo to add?” → switches to Label mode

**Label mode:**

- User takes photo of product label/box
- Show “Processing…” overlay with spinner
- Send photo to Claude API backend endpoint
- Claude returns: {name, brand, serial_number, barcode_if_visible}
- Show parsed results in editable card overlay:
  - Product Name (pre-filled, editable)
  - Brand (pre-filled, editable)
  - Serial Number (pre-filled, editable)
  - Barcode (pre-filled, editable)
  - Category (dropdown)
  - Quantity (number input)
  - Netto price (number input)
  - Brutto price (number input)
- “Save” button:
  - If context is “Add to Stock”: creates new product in database
  - If context is “Sell”: creates product + immediately creates sale record

**Shelf mode (multi-scan):**

- User takes photo of entire shelf/display
- Send to Claude API with special prompt for multi-product detection
- Claude returns array of products: [{name, brand, estimated_quantity}]
- Show scrollable list of detected products, each with checkbox and editable quantity
- User reviews, unchecks wrong detections, adjusts quantities
- “Add All to Stock” button → batch insert/update

**CRITICAL:** The scanner must feel fast. Show camera preview immediately. Process in background. Use haptic feedback on successful scan.

## BACKEND API ENDPOINTS (Express.js server in Replit)

```
POST /api/scan/label
  - Receives: base64 image
  - Sends to Claude API with prompt:
    "Analyze this product label photo. Extract and return ONLY valid JSON:
    {\"name\": \"product name\", \"brand\": \"brand name\", \"serial_number\": \"if visible or null\", \"barcode\": \"if visible or null\"}
    If you cannot identify a field, set it to null. Do not add any text outside the JSON."
  - Returns: parsed JSON to client

POST /api/scan/shelf
  - Receives: base64 image
  - Sends to Claude API with prompt:
    "Analyze this shelf/display photo. Identify ALL visible products and estimate quantities.
    Return ONLY valid JSON array:
    [{\"name\": \"product name\", \"brand\": \"brand\", \"estimated_quantity\": number}]
    Only include products you can clearly identify. Do not guess. Do not add text outside JSON."
  - Returns: parsed JSON array to client

POST /api/voice/parse-orders
  - Receives: { text: "transcribed voice text" }
  - Sends to Claude API with prompt:
    "Parse this order dictation into structured items. The text is in Russian.
    Input: {text}
    Return ONLY valid JSON array:
    [{\"name\": \"product name in original language\", \"quantity\": number}]
    If quantity is not mentioned, default to 1. Do not add text outside JSON."
  - Returns: parsed JSON array to client
```

**IMPORTANT for Claude API calls:**

- Model: claude-sonnet-4-5-20250929
- Max tokens: 1024 (enough for product data, saves cost)
- For image inputs: resize photos to max 1024px before sending (saves API cost)
- Wrap all API calls in try/catch, return manual entry form on any error
- Add request timeout of 15 seconds

1. Logout button in Reports screen header

## OFFLINE MODE

- On app start: sync all products from Supabase to local SQLite
- All reads come from local SQLite (fast)
- All writes go to BOTH local SQLite AND Supabase
- If Supabase write fails (no internet): queue the operation
- When internet returns: process queue in order
- Show small “offline” badge in header when no connection
- Use NetInfo from @react-native-community/netinfo to detect connectivity

## ERROR HANDLING

- Network errors: show toast “No connection, saved locally” and queue sync
- Claude API errors: show toast “AI unavailable, please enter manually” and show manual form
- Camera permission denied: show explanation screen with button to open settings
- Microphone permission denied: show explanation and fallback to manual text input
- Empty scan result: show “Could not recognize. Try again or enter manually”
- Supabase auth errors: show specific message (wrong password, user not found, etc.)
